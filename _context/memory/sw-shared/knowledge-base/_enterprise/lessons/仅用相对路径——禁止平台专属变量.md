# 仅用相对路径——禁止平台专属变量

**Type:** Lesson
**Created:** 2026-05-04
**Scope:** enterprise
**Author:** hw-controller

## Summary

技能（Skill）设计初期使用了 `${CLAUDE_PLUGIN_ROOT}` 等平台专属变量来引用文件路径，导致技能无法在 Codex 和 OpenCode 平台上运行。解决方案：技能中所有文件引用必须使用相对路径，禁止任何平台专属环境变量或绝对路径。

## Details

在跨平台技能设计中，技能文件需要同时运行在 Claude Code、Codex（OpenAI）和 OpenCode 三个平台上。初期设计时不小心使用了 Claude Code 专属的变量引用模式：

**错误实践**：
- 使用 `${CLAUDE_PLUGIN_ROOT}/skills/hw-tdd-agent/references/tdd-ut-cycle.md` — 只在 Claude Code 上可用
- 使用 `${CODEX_SANDBOX}/.agents/skills/...` — 只在 Codex 上可用
- 使用绝对路径如 `/home/user/.claude/skills/...` — 不同机器路径不同
- 使用跨技能遍历如 `../hw-controller/references/global-state.md` — 技能被复制到目标平台时兄弟目录可能不存在

**平台解析差异**：
- Claude Code 在版本化路径下缓存插件：`CLAUDE_PLUGIN_ROOT` 解析结果不可预测
- Codex 沙箱化技能：绝对路径指向不存在的目录
- OpenCode 从多个源头合并配置：硬编码路径在源切换时断裂
- 唯一通用的模式：`references/foo.md`、`scripts/run.sh` 从 SKILL.md 所在目录解析

**经验**：
1. 技能（Skill）的单位是自包含的目录——所有引用必须在该目录内可解析
2. 如果需要共享文件，复制到每个技能的目录中（允许冗余，不允许跨技能引用）
3. Token 模式解决项目根路径问题：使用 `{project-root}` 作为逻辑标记，每个技能在 On Activation 步骤自行解析
4. 工具调用描述意图而非名称：使用"把审查委托给安全审查 Agent"而非"使用 Agent 工具 subagent_type=hw-reviewer-security"
5. 这些规则记录在 `CLAUDE.md` 的 "Platform-Agnostic Skill Design" 和 "Working Agreement" 部分

## Context

设计跨平台 Agent 技能时，文件引用路径是首要陷阱。本教训适用于所有需要多平台部署的技能系统——设计时假设最受限的平台（Codex 沙箱环境）作为基准。
