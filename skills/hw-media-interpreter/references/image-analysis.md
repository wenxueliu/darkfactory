# 图片分析指南 (Image Analysis Reference)

当被要求分析图片文件时，按以下策略处理。根据图片类型和 call target 的目标选择对应的分析框架。

---

## 1. 图片类型识别与路由

| 图片类型 | 识别特征 | 分析重点 |
|----------|----------|----------|
| UI 截图 | 按钮、输入框、导航栏、表单、列表 | 布局结构、组件类型、文本内容、交互元素 |
| 架构图 | 方框、连线、分层、标注 | 组件、关系、数据流、依赖方向 |
| 流程图 | 菱形(决策)、矩形(步骤)、箭头(流向) | 步骤顺序、决策点、分支路径、起点终点 |
| 图表 (chart/graph) | 坐标轴、柱/线/饼、图例、标签 | 数据类型、数值、趋势、坐标轴含义 |
| 线框图/原型 | 占位符区域、线框、标注注释 | 页面分区、UI 元素位置、交互意图 |
| 数据表截图 | 行列网格、表头、单元格值 | 所有行数据、列名、数值 |
| 代码截图 | 等宽字体、语法高亮、行号 | 代码文本、文件路径（如果可见） |
| 照片/一般图片 | 自然场景、人物、物体 | 场景描述、主体识别、拍摄内容 |

---

## 2. UI 截图分析

### 2.1 结构化描述格式

当目标为"描述这个UI界面"时，使用以下结构：

```
页面类型：[登录页/列表页/详情页/表单页/仪表盘/...]

布局结构：
- 顶部导航栏：[元素描述]
- 左侧边栏：[元素描述]
- 主内容区：[元素描述]
- 底部状态栏/页脚：[元素描述]

UI 组件清单：
| 组件类型 | 位置 | 文本/标签 | 状态 |
|----------|------|-----------|------|
| 按钮 | 顶部右侧 | "提交" | 可点击(蓝色) |
| 输入框 | 中部 | placeholder:"请输入用户名" | 空 |
| 下拉菜单 | 中部 | "请选择部门" | 已展开 |
| 数据表格 | 下半部 | 5列 | 加载中 |

交互元素：
- 可点击：[列表]
- 可输入：[列表]
- 可展开/折叠：[列表]

文本内容：
[图片中所有可见文本，按从上到下从左到右的顺序]
```

### 2.2 表单页分析

```
表单字段：
| 序号 | 字段标签 | 控件类型 | 是否必填 | 验证提示 | 默认值 |
|------|----------|----------|----------|----------|--------|
| 1 | 用户名 | 文本输入框 | 是(红色星号) | "请输入用户名" | 空 |
| 2 | 密码 | 密码输入框 | 是 | "至少8位" | 空 |
| 3 | 记住我 | 复选框 | 否 | - | 未选中 |

提交按钮：位置（右下），文本（"登录"），样式（蓝色填充）
重置按钮：位置（提交按钮右侧），文本（"重置"）
```

### 2.3 列表/表格页分析

```
表格列定义：
| 列名 | 对齐 | 排序 | 备注 |
|------|------|------|------|
| 序号 | 居中 | 否 | 自动编号 |
| 名称 | 左对齐 | 是(↑) | 当前排序依据 |
| 状态 | 居中 | 否 | 彩色标签 |
| 操作 | 居中 | 否 | 操作按钮列 |

数据行（前N行作为示例）：
| 序号 | 名称 | 状态 | 操作 |
|------|------|------|------|
| 1 | xxx | 运行中(绿色) | [编辑] [删除] |
| 2 | yyy | 已停止(灰色) | [编辑] [删除] |

分页：底部分页器，"共 X 条"，当前第 Y 页
```

## 3. 架构图分析

### 3.1 分析框架

```
架构层级：
[从上到下或从外到内描述系统的分层]

组件清单：
| 组件名称 | 类型 | 所属层 | 职责 |
|----------|------|--------|------|
| API Gateway | 网关 | 接入层 | 路由、认证、限流 |
| Service A | 微服务 | 业务层 | 用户管理 |
| Redis | 缓存 | 数据层 | 会话缓存 |

组件关系（数据流/调用链）：
- API Gateway -> Service A: HTTP调用，负责路由业务请求
- Service A -> Database: 读写持久化数据
- Service A -> Redis: 读写缓存数据
- Service A -> Message Queue: 发布事件通知

外部系统/用户：
- 客户端: 通过 HTTPS 访问 API Gateway
- 第三方支付: 通过回调接口与 Service B 交互

协议/通信方式：
- 内部服务间: gRPC (同步调用)
- 异步消息: Kafka (事件驱动)
- 外部接口: REST/HTTPS

关键设计决策（如果图中标注了）：
- 为什么选择这个架构模式
- 数据流向的考虑
- 扩展点和瓶颈
```

### 3.2 部署架构图分析

额外关注：
- 物理节点/容器数量和分布
- 网络分区（VPC/子网/安全组）
- 负载均衡配置
- 副本数量和高可用设计
- 持久化存储方案

### 3.3 类图/ER 图分析

```
实体/类清单：
| 类名 | 类型 | 关键属性 | 关键方法 |
|------|------|----------|----------|
| User | Entity | id, name, email | authenticate() |
| Order | Entity | id, userId, total | calculateTotal() |

关系：
- User 1---N Order: 一个用户有多个订单（一对多）
- Order N---N Product: 订单和产品多对多 (通过 OrderItem)
```

## 4. 流程图分析

### 4.1 分析框架

```

终点：[结束节点]

主流程步骤：
1. [步骤1描述] -> [步骤2描述] -> [步骤3描述] -> ...

  - 否/拒绝 -> [路径B]

  - 分支1 -> [路径C]

- [异常场景]: 流程中断/回退/人工处理

关键参与者（如果有泳道）：
```

## 5. 图表 (Charts/Graphs) 分析

### 5.1 柱状图/条形图

```
图表标题：[title]

X轴：[轴标签]，取值：[值1, 值2, 值3, ...]
Y轴：[轴标签]，范围：[min] ~ [max]，单位：[unit]

数据系列：
| X值 | Y值 |
|-----|-----|
| A   | 100 |
| B   | 250 |
| C   | 180 |

趋势/模式：
- 最高值：X=B, Y=250
- 最低值：X=A, Y=100
- 趋势方向：先升后降

图例（如有多系列）：
- 蓝色：系列1名称
- 橙色：系列2名称
```

### 5.2 折线图

```
[同上取X/Y轴信息]

趋势分析：
- 整体走势：[上升/下降/平稳/波动]
- 关键拐点：X=3处开始上升，X=7处达到峰值
- 增长率：从X=3到X=7，值从50增长到200，增幅300%

多线比较（如有多条线）：
- 系列A始终高于系列B
- 系列A和C在X=5处交叉
```

### 5.3 饼图/环形图

```
总计：[total value]

各项占比：
| 类别 | 数值 | 占比 |
|------|------|------|
| A    | 100  | 25%  |
| B    | 200  | 50%  |
| C    | 100  | 25%  |

最大份额：B (50%)
```

### 5.4 仪表盘/可视化面板

```
面板标题：[dashboard title]

指标卡片：
| 指标 | 当前值 | 环比/同比 | 趋势 |
|------|--------|-----------|------|
| 总用户数 | 12,345 | +15% | ↑ |
| 活跃率 | 68.2% | -3% | ↓ |

图表区域分布：
- 左上：折线图 - 日活趋势
- 右上：柱状图 - 收入分布
- 下方：表格 - Top 10 用户
```

## 6. 文本提取（图片中的文字）

当目标是提取图片中的所有文字时：

1. 按从上到下、从左到右的顺序描述
2. 保持文本的相对位置关系
3. 注明文字样式（加粗、字号、颜色、对齐方式）
4. 对于代码截图，尽可能还原原始代码文本

```
提取的文本内容：

[顶部标题栏]
"系统管理平台" (白色, 16px, 居中)

[左侧导航]
"首页"
"用户管理"
"系统设置"

[主内容区 - 表单]
"用户名：" [输入框]
"密码：  " [输入框]
[提交按钮] "登录"
```

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 图片无法加载 | 返回："无法读取图片 {path}：文件格式不支持或已损坏" |
| 图片内容无法识别 | 返回："图片内容无法识别：{原因（模糊/低对比度/非标准格式等）}" |
| 目标信息不在图片中 | 返回："未找到{目标信息}：图片中不包含该内容。图片实际展示的是：{简要描述}" |
| 图片过于模糊 | 尽最大努力描述可辨识的部分，标注不可辨识的部分 |
