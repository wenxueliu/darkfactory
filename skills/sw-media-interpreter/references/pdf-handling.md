# PDF 文件处理指南 (PDF Handling Reference)

当被要求解读 PDF 文件时，按以下策略处理。根据 call target 的具体目标选择对应策略。

---

## 1. PDF 页面导航与选择

大 PDF 文件无法一次读取全部内容时，使用分段读取策略：

### 1.1 策略一：按页码范围读取

```
Read file_path="/path/to/doc.pdf" pages="1-5"   # 先读取前5页了解结构
Read file_path="/path/to/doc.pdf" pages="6-10"  # 继续读取目标章节
```

**适用场景：** 目标章节位置不明确，需先探索文档结构。

### 1.2 策略二：直接目标章节读取

如果 call target 已指明具体章节或页码范围，直接读取对应页面：
```
目标: "提取第3章内容" → pages="包含第3章的页码范围"
目标: "提取5.2节" → 先定位目录，再读取对应页
目标: "提取附录A" → pages="附录A所在页"
```

### 1.3 策略三：关键词定位

当目标为特定主题但未指明位置时：
1. 先读取目录页（通常在前5页）
2. 根据目录定位目标章节的页码范围
3. 读取对应页面
4. 如果目录不可用，按 5 页为单位扫描，寻找目标关键词

## 2. 文档结构识别

### 2.1 标题层级识别

根据字号、加粗、编号格式识别文档结构：

```
识别特征：
- 一级标题（H1）：最大字号、居中或左对齐、带章节编号如"第X章"
- 二级标题（H2）：次大字号、带编号如"X.Y"或"X.Y.Z"
- 三级标题（H3）：再次字号、带编号如"X.Y.Z.W"

输出格式：
## 第1章 概述
### 1.1 背景
### 1.2 目标
```

### 2.2 目录提取

当目标为"提取目录"时：
1. 读取前3-5页
2. 识别所有带章节编号的行
3. 输出结构化的目录列表：

```
第1章 概述 (p.1)
  1.1 背景与目标 (p.1)
  1.2 适用范围 (p.3)
  1.3 术语定义 (p.4)
第2章 系统架构 (p.8)
  2.1 总体架构 (p.8)
  2.2 模块划分 (p.12)
...
```

### 2.3 元数据提取

当目标涉及文档元数据时，关注：
- 文档标题（封面页最大字号文本）
- 作者/部门（封面或页眉）
- 版本号/日期（封面或页脚，格式如"v1.2"、"2024-01-15"）
- 文档编号（如"DOC-2024-001"）
- 密级标记（如"内部"、"机密"、"绝密"）

## 3. 表格解析

### 3.1 识别表格

PDF 中的表格识别特征：
- 规则排列的行列结构
- 单元格之间的对齐关系
- 表头行（通常加粗或带背景色）
- 边框线或颜色交替行

### 3.2 简单表格（≤5列）

输出为 Markdown 表格：

```
| 序号 | 字段名 | 类型 | 必填 | 说明 |
|------|--------|------|------|------|
| 1    | name   | string | 是 | 用户名 |
| 2    | age    | int   | 否  | 年龄 |
```

### 3.3 复杂表格（跨行/跨列）

对合并单元格，注明合并范围：

```
| 项目 | Q1 | Q2 | Q3 | Q4 |
|------|----|----|----|----|
| 收入 | 100 | 120 | [Q3-Q4 合并] 280 |
| 成本 | 80  | 85  | 90 | 95  |
```

### 3.4 宽表格（>5列）

使用转置或分组描述：

```
字段列表（横向展示）：
  1. name       | 类型: string | 必填: 是 | 说明: 用户名
  2. age        | 类型: int    | 必填: 否 | 说明: 年龄
  3. email      | 类型: string | 必填: 是 | 说明: 邮箱地址
```

## 4. 数据提取模式

### 4.1 提取特定章节内容

```
目标: "提取第3章 系统设计的所有内容"

做法:
1. 定位第3章的页码范围（通过目录或扫描）
2. 只读取该章所在页面
3. 提取该章所有文本内容
4. 保持原有层级结构
5. 不包含前后章节的内容
```

### 4.2 提取特定类型信息

```
目标: "提取文档中所有的 API 端点"

做法:
1. 识别 API 端点的特征模式（如 GET/POST /api/...）
2. 扫描全文（或目标章节）
3. 收集所有匹配项
4. 输出为列表：

API 端点列表：
- GET  /api/v1/users       | 第3章 | 获取用户列表
- POST /api/v1/users       | 第3章 | 创建新用户
- GET  /api/v1/users/{id}  | 第3章 | 获取用户详情
```

### 4.3 提取配置参数

```
目标: "提取所有的配置参数和默认值"

做法:
1. 寻找配置相关章节（通常标题含"配置"、"参数"、"设置"）
2. 识别参数格式：参数名、类型、默认值、取值范围、说明
3. 输出结构化配置清单
```

### 4.4 提取文本片段

```
目标: "找到描述'认证流程'的段落"

做法:
1. 搜索关键词"认证"
2. 定位到包含关键词的段落
3. 提取完整的段落内容（从上一段结束到下一段开始之间的所有文本）
4. 如有跨页，合并连续内容
5. 输出时注明所在章节和页码
```

## 5. 扫描型 PDF 处理

对于扫描件（图片型 PDF），Read 工具可读取但精度取决于扫描质量：

1. **质量好的扫描件：** 直接读取，提取文本
2. **质量差的扫描件：** 如实描述可辨识的内容，标注不可辨识部分："[该区域文本无法辨识]"
3. **混合型 PDF：** 可能部分页面是文本、部分是扫描件，逐页判断

## 6. 常见 PDF 提取模式速查

| 目标类型 | 做法 | 输出格式 |
|----------|------|----------|
| 全文摘要 | 读取全文，按章节提炼要点 | 结构化摘要（按章-节） |
| 目录 | 读取前5页，识别章节标题 | 缩进目录列表 + 页码 |
| 表格数据 | 定位表格所在页，解析单元格 | Markdown 表格 |
| 特定章节 | 只读目标章节所在页 | 该章节完整文本 |
| 特定信息 | 全文扫描匹配关键词 | 键值对列表 |
| 配置参数 | 定位配置章节 | 参数-类型-默认值-说明 |
| 流程图描述 | 读取图表所在页，描述文本 | 结构化流程描述 |
| 代码片段 | 定位代码块区域 | 代码块 + 所在章节 |

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 文件不存在 | 返回："无法读取文件 {path}：文件不存在" |
| 文件无法打开 | 返回："无法读取文件 {path}：{错误信息}" |
| PDF 已加密/需要密码 | 返回："无法读取文件 {path}：PDF 文件已加密，需要密码" |
| 目标章节不存在 | 返回："未找到{目标章节}：文档中不包含此章节。可用章节：[列出实际章节]" |
| 目标信息不存在 | 返回："未找到{目标信息}：{解释为什么找不到}" |
| 表格解析失败 | 返回："表格解析失败：{原因}。原始表格内容：[尽可能输出模糊版本]" |
