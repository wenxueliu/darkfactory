## 4. 用户旅程映射

用户故事描述的是"谁要什么"，用户旅程描述的是"从开始到结束，用户经历什么"。这是连接用户故事和验收标准的桥梁——验收标准必须覆盖用户旅程的每个关键阶段。

### 4.1 旅程阶段

```
{阶段1} → {阶段2} → {阶段3} → {阶段4} → {阶段5}
    │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼
  [用户做什么?] [系统响应?] [用户感受?] [哪里会卡?] [成功是什么?]
```

| 阶段 | 用户行为 | 触点 | 情感状态 | 痛点/机会 | 成功指标 |
|------|---------|------|---------|----------|---------|
| {阶段1: 如: 发现} | {行为} | {App/Web/API/CLI} | {期待/焦虑/困惑} | {当前哪里不好} | {怎么衡量} |
| {阶段2} | {行为} | {触点} | {情绪} | {痛点} | {指标} |
| {阶段3} | {行为} | {触点} | {情绪} | {痛点} | {指标} |

**最少要求:** 至少覆盖 3 个阶段。对于纯后端/API 需求，用"调用方视角"代替"用户视角"——API 消费者怎么发现、调用、处理错误、重试。

### 4.2 关键交互时刻 (Moments of Truth)

用户在哪些节点会做出"继续用还是放弃"的判断:

| 关键时刻 | 用户期望 | 如果失败... | 对应 US |
|---------|---------|------------|--------|
| {如: 首次加载} | {秒开} | {用户流失} | US-{N} |
| {如: 支付确认} | {秒级响应 + 明确反馈} | {重复支付/放弃购买} | US-{N} |

### 4.3 异常路径

```
正常路径: A → B → C → D → 完成

异常路径:
  A → (超时) → 重试 → B
  B → (校验失败) → 提示 → A
  C → (外部服务不可用) → 降级 → D'
  D → (权限不足) → 引导登录 → D
```
